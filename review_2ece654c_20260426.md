## Finding: All Artifact Sources Are Code-Empty

### Primary Repository (mail-research/lrm-critique-vectors)

The repository at `https://github.com/mail-research/lrm-critique-vectors` contains only:

1. `LICENSE` — GPLv3 license text (35,149 bytes)
2. `.gitignore` — standard Python gitignore template
3. Single commit (`89de30e`, "Initial commit")

## What Cannot Be Reproduced

The paper's central claims are entirely unsupported by the linked artifact:

1. **Error Injection Framework (Section 3.1):** The paper uses GPT-5 with a carefully engineered prompt $P_{err}$ to inject arithmetic errors into GSM8K and MATH500 reasoning traces to construct GSM8K-Error and MATH500-Error. No prompt, no generation script, no dataset construction code.

2. **Feature Space Analysis (Section 4.2, Table 2):** Linear probing experiments on residual stream activations across 4 model families (Qwen3-4B, R1-8B, R1-14B, R1-32B) with near-perfect AUROC. No probe training code, no activation extraction pipeline, no evaluation script.

3. **Critique Vector Extraction (Section 4.3, Eq. 5):** Difference-in-means computation between intervened and baseline activations. No vector extraction code, no activation data, no training dataset construction.

4. **Steering Experiments (Section 4.4, Figures 3-4):** Steering coefficient sweeps ($\alpha \in [-1, 1]$) evaluated on ProcessBench and BIG-Bench Mistake. No steering implementation, no evaluation harness, no result generation scripts.

5. **Logit Lens Interpretation (Section 4.5, Table 3):** Vocabulary projection of critique vectors. No unembedding projection code, no token ranking script.

6. **Test-Time Scaling (Section 5.3):** Budget forcing experiments with critique vector steering. No budget forcing implementation, no scaling evaluation code.

7. **Model Infrastructure:** No model loading scripts for Qwen3-4B, R1-8B, R1-14B, or R1-32B. No tokenizer configuration. No generation parameter specification.

## Impact on Reproducibility

This is not an inference-complete-but-training-incomplete release. This is a completely empty placeholder — the repository exists but contains no implementation whatsoever. Every empirical claim in the paper (error recovery rates, probing AUROC, steering coefficient effects, logit lens tokens, test-time scaling results) is unreproducible from the linked artifact. The paper's core contribution — extracting and steering a critique vector from latent space — cannot be independently verified.

### Tarball (Paper Source Archive)

The tarball at `2ece654c-b390-41a8-a849-46254082efab.tar.gz` contains only LaTeX source files (`arxiv.tex`, `main.bib`, ICML style files, figures as PDFs), plus qualitative example appendices. **No Python code, scripts, data, or implementation artifacts.**

### Benchmark Repos (ProcessBench, BIG-Bench-Mistake)

`QwenLM/ProcessBench` and `WHGTyen/BIG-Bench-Mistake` are legitimate upstream benchmark repositories containing evaluation data and baseline code. However, they do not contain any of the paper's novel implementation: error injection, activation extraction, linear probing, critique vector computation, steering, or test-time scaling pipeline. They are evaluation targets, not the paper's method implementation.
