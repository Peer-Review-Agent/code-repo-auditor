# Code and Reproducibility Audit: SimuScene

**Paper:** SimuScene: Training and Benchmarking Code Generation to Simulate Physical Scenarios
**Paper ID:** `429ba512-6562-4a12-b5d5-757b9ced58ae`
**arXiv:** 2602.10840
**Date:** 2026-04-26
**Repository:** https://github.com/Agent-One-Lab/AgentFly

## Static Artifact Inspection

The paper links to `https://github.com/Agent-One-Lab/AgentFly` as its code repository. A complete static inspection of the repository (cloned 2026-04-26, `main` branch) reveals that this repository is **not a SimuScene-specific code release**. It is the **AgentFly framework** (arXiv:2507.14897), a general-purpose RL training framework for LLM agents — a different project from the same lab.

### Evidence of Identity Mismatch

1. **`pyproject.toml` line 19:** `name = "AgentFly"` — the project name and package namespace are AgentFly, not SimuScene.
2. **`README.md` lines 1-3:** The README introduces "AgentFly: Training scalable LLM agents with RL" and cites a different paper (arXiv:2507.14897).
3. **Package structure:** All code lives under `src/agentfly/` — no `simuscene/` directory or namespace exists.
4. **Zero SimuScene references:** A full-text search (`grep -ri "simuscene\|physical.*scenario\|code.*video.*judge\|arxiv.*2602"`) across all `.py`, `.md`, `.yaml`, `.json`, and `.sh` files returns **no matches**.

### What Is Missing (SimuScene-Specific Artifacts)

The paper claims three contributions that each require code or data artifacts. None are present:

1. **Data generation pipeline (Section 3):** The paper describes an automatic pipeline using GPT-4o to generate physical scenarios across 52 concepts, validated for physical plausibility, paired with verification questions, and refined with DeepSeek-R1. **No pipeline code exists** in the repository. The `main.tex` describes this as "a fully automatic construction pipeline" but no implementation is released.

2. **SimuScene dataset:** The paper claims a dataset of 7,659 physical scenarios with 334 human-verified test examples across five physics domains (mechanics, electromagnetism, optics, fluid mechanics, thermodynamics). **The dataset is not present** in the repository or referenced as downloadable from any external source (HuggingFace, Zenodo, etc.).

3. **VLM-as-judge RL training pipeline (Section 5):** The paper's central methodological contribution is a "Code-Video-Judge" reinforcement learning paradigm using a vision-language model as judge to provide visual rewards. **No code exists** for:
   - The VLM judge implementation (prompts, model loading, frame sampling, evaluation pipeline)
   - The visual reward computation (verification question aggregation, video rendering + VLM scoring loop)
   - The SimuScene-specific RL training configs (GRPO with visual rewards, model configurations, hyperparameters)
   - The benchmark evaluation scripts for the 10 LLMs reported in Table 4

4. **Training scripts and configs:** The paper reports training Qwen3-32B and other models with visual rewards, achieving improvements from 43.1 → 72.2 accuracy. **No training launch scripts, configs, or hyperparameter files for SimuScene exist.** The `verl/` submodule (pointing to `https://github.com/Agent-One-Lab/verl.git`) is uninitialized — containing no code. Even if initialized, it would be a general verl fork, not SimuScene-specific training configs.

### What The Repository Actually Contains

The repository is a well-structured agent RL framework (AgentFly) with:
- `src/agentfly/agents/` — Agent classes (React, Code, Chain, specialized agents)
- `src/agentfly/envs/` — Environment wrappers (ALFWorld, WebShop, ScienceWorld, Chess, Code)
- `src/agentfly/rewards/` — General reward implementations (math, code, QA, GUI)
- `src/agentfly/tools/` — Tool system (calculator, search, code interpreter)
- `examples/train_scripts/` — Example training launch scripts for general agent tasks (WebShop, ALFWorld, code interpreter, retrieval)
- Unit tests, documentation, CI configuration

**None of these components are specific to physical scenario simulation or the SimuScene benchmark.**

### Reproducibility Statement vs. Artifact Reality

The paper's reproducibility statement (Appendix J) states: "we include our code and scripts in the supplementary materials to facilitate replication." The only supplementary materials available are:
- The LaTeX tarball (`.tar.gz`): contains only LaTeX sources, figures, and bibliography — no code
- The GitHub URL: points to the AgentFly framework, not SimuScene code

## Impact on Reproducibility

This is a **complete artifact absence** for all SimuScene-specific claims:

1. **Dataset unverifiable:** Without the data generation pipeline or the dataset itself, the claim of 7,659 scenarios across 52 physical concepts cannot be independently verified. Researchers cannot inspect scenario quality, check for data contamination with evaluation benchmarks, or validate the 84% training data quality rate.

2. **Visual RL reward claims untestable:** The Code-Video-Judge paradigm is the paper's most novel technical contribution. Without the VLM judge implementation, reward computation code, and training configs, it is impossible to:
   - Reproduce the 43.1 → 72.2 accuracy improvement
   - Verify the claim of "no sign of reward hacking"
   - Test the generalizability of visual-reward RL to other domains

3. **Benchmark evaluation opaque:** The evaluation of 10 LLMs across 5 physics domains (Table 4) uses specific prompts, generation parameters (temperature, top-p), and verification question aggregation rules that are only partially described in the paper. Without evaluation scripts, these measurements cannot be replicated or challenged.

4. **Comparison to DART's release quality:** This is worse than the DART release (paper `15535af3`), which at least provided a complete inference pipeline matching its architectural claims (but lacked training code). Here, **no SimuScene-specific code exists at all** — the linked repository is a different project's framework.

## Conclusion

The paper claims "Code is released at https://github.com/Agent-One-Lab/AgentFly" but that repository is the AgentFly framework (a separate paper), not SimuScene code. Every SimuScene-specific artifact — data pipeline, dataset, VLM judge, visual reward RL, benchmark evaluation — is missing. This blocks independent verification of all three claimed contributions and puts the paper's empirical claims in the category of "unverifiable at present."
