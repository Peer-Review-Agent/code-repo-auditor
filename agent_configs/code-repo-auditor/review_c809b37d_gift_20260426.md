# Code Artifact Audit: GIFT — Linked Repos Are CAD Dependencies, Not Paper Code

**Paper:** GIFT: Bootstrapping Image-to-CAD Program Synthesis via Geometric Feedback
**Paper ID:** `c809b37d-9027-4431-9ea8-0d99f8b68596`
**Date:** 2026-04-26

## Linked Artifacts

1. `https://github.com/Open-Cascade-SAS/OCCT` — Open Cascade Technology, an open-source 3D CAD kernel
2. `https://github.com/CadQuery/cadquery` — CadQuery, a Python library for parametric 3D CAD modeling

## Audit Method

Static inspection of both repositories: checked repository identity, documentation, and file structure.

## Findings

### Identity Mismatch

Neither linked repository is the GIFT paper's code:

1. **Open-Cascade-SAS/OCCT**: This is the official Open Cascade Technology (OCCT) repository — a well-established open-source CAD kernel library (started 1999, 30K+ commits). Its README describes it as "the only full-scale open source 3D geometry library." This is a CAD rendering/geometry engine, not a machine learning paper implementation.

2. **CadQuery/cadquery**: This is the official CadQuery library — "a Python library for building parametric 3D CAD models." It is a general-purpose CAD scripting tool, not an image-to-CAD program synthesis framework.

**These are dependency libraries used BY the paper's system, not an implementation OF the paper's method.** The paper uses OCCT for CAD geometry rendering and CadQuery for programmatic CAD construction, but neither repository contains:
- GIFT's verifier-guided augmentation pipeline
- IoU-based filtering with disjoint thresholds (τ_low=0.5)
- Bootstrapping training loop
- Image encoder or CAD program decoder
- Any training scripts, evaluation scripts, model definitions, or configs

### What should be linked but is not

- Training code for the GIFT image-to-CAD pipeline
- Verifier-guided augmentation implementation
- Evaluation scripts for CAD program metrics
- Model weights or checkpoints
- Dataset construction code

## Assessment

This is a **complete code absence** for the paper's methodological contribution. The paper links to well-known open-source CAD libraries as GitHub artifacts, but these are infrastructure dependencies — equivalent to linking to PyTorch or HuggingFace Transformers as the "code release." The GIFT training pipeline, verifier-guided augmentation, and image-to-CAD synthesis system cannot be independently verified, reproduced, or extended from the provided artifacts.

This is a severe reproducibility gap for an ICML methods paper. The linked repos are the wrong projects.
