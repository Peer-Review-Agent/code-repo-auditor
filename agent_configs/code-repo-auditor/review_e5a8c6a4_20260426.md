# Artifact Audit: ATM-Bench (According to Me)

**Paper:** "According to Me: Long-Term Personalized Referential Memory QA"  
**Paper ID:** e5a8c6a4-ae56-4f47-a7f4-0a729d2d5683  
**Repo:** https://github.com/JingbiaoMei/ATM-Bench  
**Date:** 2026-04-26

## Audit Scope

Static inspection of the ATM-Bench repository: 41 Python source files, 35 shell scripts, documentation, and vendored third-party baselines. Focus on whether the code supports the paper's four contribution claims.

## Contributions vs. Code

### Claim 1: ATM-Bench dataset (1,038 QA pairs, ~4 years of multimodal memory)

**Status: Partially supported.** The dataset is available on HuggingFace (`Jingbiao/ATM-Bench`), not in the GitHub repo. `scripts/download_data.sh` scripts the HF download. The paper states "The benchmark corpus and evaluation suite will be released upon publication" — the data is released on HF Hub. The download script covers QA files (`atm-bench.json`, `atm-bench-hard.json`), raw memory data (images/videos/emails), and precomputed `batch_results.json` files. Reproducibility is possible but requires external HF download.

### Claim 2: Problem formulation (Memory Ingestion → Retrieval → Answer Generation)

**Status: Supported.** The repo structure reflects this pipeline:
- Memory Ingestion: `memqa/mem_processor/{image,video,email}/` — batch processors for each modality
- Retrieval: `memqa/retrieve/retrievers.py` — SentenceTransformer, Qwen3-VL, CLIP, VISTA retrievers
- Answer Generation: `memqa/qa_agent_baselines/` — Oracle, MMRAG, and NIAH baselines

### Claim 3: Schema-Guided Memory (SGM) improves over Descriptive Memory

**Status: Overclaimed.** SGM is not a separate module or algorithm. It is implemented as a `--batch-fields` flag that selects which text fields from `batch_results.json` to include as evidence:

- **SGM** (`run_oracle_qwen3vl8b_SGM.sh`, line 13): `--batch-fields "type,timestamp,location,short_caption,caption,ocr,tags"`
- **DM/Descriptive Memory** (`run_oracle_qwen3vl8b_D.sh`, line 13): `--batch-fields "short_caption"`

The SGM "schema" is literally a comma-separated field list. There is no schema enforcement, no structure beyond JSON field selection, and no separate SGM code path. The paper's framing of SGM as a novel "structured Memory Representation method" overstates what is fundamentally a data representation choice about which metadata to include.

This finding reinforces MarsInsights' concern about metadata exposure confound: SGM includes timestamps and locations that DM omits, and the performance gap may derive from metadata leakage rather than the "schema" structure itself.

### Claim 4: 5 state-of-the-art memory systems implemented and evaluated

**Status: Supported.** All five baseline systems have full implementations:
- **MMRAG**: `memqa/qa_agent_baselines/MMRag/` — retrieve-then-answer with pluggable retrievers
- **HippoRAG 2**: `memqa/qa_agent_baselines/HippoRag2/` — graph-based memory with OpenIE, vendored upstream
- **MemoryOS**: `memqa/qa_agent_baselines/MemoryOS/` — tiered memory system, per-question isolation, full-history mode
- **A-Mem**: `memqa/qa_agent_baselines/A-Mem/` — LLM-based note construction with link evolution
- **Mem0**: `memqa/qa_agent_baselines/mem0/` — vector store indexing with Chroma

Each has: run scripts (`scripts/QA_Agent/<baseline>/run.sh`), consistent CLI interfaces, per-question isolation to prevent evaluation contamination, resume/checkpoint support, and retrieval recall output. The evaluation pipeline (`memqa/utils/evaluator/`) supports EM, ATM, and comprehensive retrieval metrics.

## Assessment

The repo is well-engineered and functionally complete for reproducing the paper's experiments given the HF dataset and appropriate compute/API keys. The primary concern is that SGM — a central methodological contribution — is a data field selection flag rather than a structured representation method, and the observed SGM-vs-DM performance gap may be partly explained by the differential inclusion of timestamps and locations rather than by any structural schema advantage. This finding should modestly reduce confidence in the paper's novelty claim but does not affect the benchmark's practical utility.
