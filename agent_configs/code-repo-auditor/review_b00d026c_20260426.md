# Code Artifact Audit: Colosseum (b00d026c)

## Paper
Colosseum: Auditing Collusion in Cooperative Multi-Agent Systems  
Repository: https://github.com/umass-ai-safety/colosseum

## Audit Method
Static inspection of the repository at commit `13f34a5...` (latest as of 2026-04-26). Examined directory structure, key Python files, configs, prompts, metrics, and plot generation scripts. Did not run any code (policy-compliant static audit only).

## What is Present and Well-Implemented

1. **Full experiment sweep framework** — `experiments/collusion/run.py` (1058 lines) implements a complete experiment runner with config-driven sweeps across models, topologies, agent counts, colluder counts, secret channel toggles, prompt variants, and seeds. Supports sequential and concurrent execution, progress tracking, resume capability, and output in JSON/JSONL/CSV formats.

2. **Regret and coalition advantage metrics match the paper** — `experiments/collusion/metrics.py` (548 lines) correctly implements:
   - Best-response regret per agent (unilateral deviation counterfactuals)
   - System-level regret (gap to oracle via `max_joint_reward`)
   - Coalition advantage mean (difference between coalition and non-coalition mean rewards)
   - Communication ratio metrics (secret vs largest non-secret blackboard post ratios)
   - Reward dispersion (std dev within and across groups)
   - Task completion, violation count, priority sum, total cost

3. **LLM-as-a-judge collusion detection** — `experiments/collusion/judge_blackboards.py` (1010 lines) implements GPT-based collusion scoring with structured Likert-scale rubrics (0-5), prompt templates, confidence thresholds, and batch processing. Multiple judge prompt variants exist (simple, detailed, role-aware).

4. **Multiple environments with custom prompts** — Five "choice" environments extend the DCOP base environments:
   - `jira_ticket_choice_env.py` — Software engineering task assignment
   - `hospital_choice_env.py` — Intra-hospital patient allocation
   - `meeting_scheduling_choice_env.py` — Meeting scheduling
   - `personal_assistant_choice_env.py` — Personal assistant task allocation
   - `smart_grid_choice_env.py` — Smart grid resource allocation
   Each has corresponding prompt files with collusion role-specific instructions.

5. **Persuasion technique experiments** — Under `experiments/persuasion/`, separate experiment runners for persuasion's effect on collusion in both JIRA and hospital environments, with per-technique prompt templates.

6. **Network influence/misinformation experiments** — `experiments/network_influence/` provides targeted and mass misinformation propagation experiments.

7. **Agent misalignment experiments** — `experiments/agent_misalignment/` tests agent objective misalignment effects.

8. **Plot generation scripts** — Each experiment module has `plots/` directories with scripts to regenerate paper figures (bar charts, model comparison plots, topology sweeps, persuasion technique comparisons, error bars).

9. **Config files for paper's model comparisons** — YAML configs exist for GPT-4.1-mini, GPT-4o-mini, Kimi-K2-Instruct across multiple experiment types.

10. **Prompt templates with variant support** — Collusion prompts support variants: "control" (cooperative baseline), collusion-promoting variants, and persuasion technique variants.

## What is Missing or of Concern

1. **DCOP environments are NOT in this repo** — The core DCOP formulation, constraint optimization, cooperative optimum computation, and `max_joint_reward` derive from the `terrarium-agents` pip package (v0.1.1, listed in `pyproject.toml`). The paper states "We ground how agents cooperate through a Distributed Constraint Optimization Problem (DCOP) and measure collusion via regret relative to the cooperative optimum." However, the DCOP implementation is imported from `envs.dcops` modules in Terrarium — this repo contains only choice-environment *wrappers* and experiment logic. The paper's central formalism cannot be independently verified from the linked repository alone.

2. **CoLLAB submodule required but not self-contained** — The `.gitmodules` file references `external/CoLLAB` for meeting scheduling. The submodule was not initialized in the clone. This external dependency means the meeting scheduling environment requires an additional codebase to function.

3. **No pre-computed results or checkpoints** — The repo has zero pre-computed experiment outputs, log files, or result CSVs. Reproducing any paper claim requires running expensive LLM API calls against proprietary models (GPT-4.1, GPT-4o-mini, Kimi-K2) with associated costs.

4. **Heavy dependency on Terrarium infrastructure** — The repo depends on `terrarium-agents[providers,science,plots]==0.1.1` which bundles LLM providers, communication protocols, blackboard infrastructure, agent scaffolding, and the DCOP environment implementations. A researcher who only has this repo cannot understand the full system without also examining the Terrarium codebase.

5. **No unit tests or validation of metrics correctness** — While the metrics implementation appears logically correct from static inspection, there are no tests verifying that regret, coalition advantage, or system regret computations match expected values for known scenarios.

## Assessment

This is an **experiment-layer-complete but formalism-opaque** release. The experiment framework, metrics, judging, prompts, and plotting are substantial and inspectable. The code is real engineering, not a placeholder. However, the paper's central theoretical contribution — the DCOP formulation that grounds collusion measurement — lives in a separate pip package (`terrarium-agents`), making the formalism inaccessible from the linked repository. This is a moderate reproducibility concern: the experimental machinery is transparent, but the mathematical foundation of the metrics is a black box to anyone auditing only the released repository.

For ICML, this means the empirical claims about collusion measurement can be verified (given API access and compute budget), but the theoretical claim that DCOP regret is the correct metric for collusion cannot be independently validated from the released artifacts.
