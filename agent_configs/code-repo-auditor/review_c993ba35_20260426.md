## Findings

### What is present and correctly implemented

1. **Alternating G-LEARN/L-LEARN structure** — `alternating_marl.py:266-321`: The `train()` method alternates G-LEARN (`g_learn()`, line 285) and L-LEARN (`l_learn()`, line 288) with convergence checking and revert-to-best mechanism, matching the paper's algorithmic description.

2. **Subsampled count-vector state** — `alternating_marl.py:231-234`: The evaluation loop subsamples k agents and constructs count vectors via `np.bincount`, matching the paper's observability constraint.

3. **Global agent value iteration** — `global_agent_optimizer.py:179-195`: Deterministic mean-field Bellman backup with vectorized Q-value computation, correctly implementing the (s_g, count_vector) MDP formulation.

4. **Function approximation variant** — `training.py` provides a continuous-state extension with policy-gradient agents, including mode-tracking accuracy as a diagnostic metric.

### What is MISSING (blocks verification of paper claims)

1. **No multi-robot control experiments.** The paper's abstract claims "we validate our results in numerical simulations for multi-robot control and federated optimization." The repository contains:
   - A discrete 5×5 toy grid environment (`marl_example.py`) with synthetic transitions
   - A continuous warehouse environment (`continuous_env.py`) with Gaussian dynamics
   - **Zero multi-robot control code** — no robot dynamics, no robot environments, no multi-robot benchmarks
   - **Zero federated optimization code** — no federated learning setup, no client-server architecture, no non-IID data partitioning
   
   A full-text grep for "robot", "federated", "warehouse", "control" confirms: the paper's motivating applications are entirely absent from the released artifacts.

2. **Algorithm-class mismatch.** The paper describes the global agent performing "subsampled mean-field Q-learning." However:
   - The tabular implementation uses **model-based value iteration** (requires known transition model P_g, P_l passed as callables at line 55-58 of `alternating_marl.py`), not model-free Q-learning
   - The function approximation variant uses **supervised cross-entropy** for the global agent (`training.py:56-84`: `F.cross_entropy(logits, label_t)` trained on (histogram, true_mode) pairs) and **REINFORCE** for the local agent — neither is Q-learning
   
   This is a material mismatch: Q-learning implies the agent learns from environment interaction without a known transition model, but the tabular implementation requires the full model, and the function approximation implementation trains from supervised labels.

3. **Experiment scale does not match paper claims.** `hyperparameters.json` shows:
   - 5 global states, 5 local states (toy problem)
   - 5 outer iterations (`n_outer_iterations: 5`)
   - Horizon 50, 50 evaluation rollouts
   - The paper claims results for "large-scale platforms and networked control systems" but the code operates on the smallest possible discrete MDP

4. **No benchmark comparisons.** There are zero baseline algorithms implemented. The paper claims O(1/√k) approximation improvements, but no baseline MARL algorithms (independent Q-learning, centralized critic, COMA, QMIX, MADDPG) are present for comparison.

5. **No Nash equilibrium distance measurement.** The code measures cumulative discounted reward and mode-tracking accuracy, but the paper's central theoretical claim (O(1/√k) approximate Nash Equilibrium) requires measuring distance to Nash equilibrium — no such metric is computed.

6. **No evaluation scripts for paper figures/tables.** The repository produces reward-vs-k curves but has no scripts aligned with any specific paper table or figure. No result CSVs, no LaTeX table generation, no statistical test scripts.

### Assessment

This is a demo-scale, theory-mismatched release. The alternating best-response structure is faithfully implemented, but the code operates at toy scale (5-state MDPs) with simplified variants of the claimed algorithm (model-based VI instead of Q-learning, supervised cross-entropy instead of RL). The paper's motivating applications (multi-robot control, federated optimization) have zero code support. The central theoretical claim (O(1/√k) Nash equilibrium) cannot be empirically verified from the released artifacts because no Nash distance metric is computed. This is a more severe artifact gap than typical training-artifact incompleteness: the code not only lacks reproduction artifacts but also deviates from the paper's claimed algorithm class and entirely omits the motivating domains.
