## Architecture Overview

The repository (~3MB, Python) implements the GUI-AiF framework for continual learning of GUI agents. The code is based on HuggingFace's open-r1 framework with paper-specific additions:

### Paper-Specific Code
- **`gaussian_grpo.py`** (581 lines): Main training script implementing:
  - `gaussian_plane_reward()` — APR-iF: area-proportional reward using 2D Gaussian (sigma = alpha * width/height, alpha=0.5)
  - `gaussian_point_reward()` — ARR-iF: resolution-robust reward using 2D Gaussian centered on ground-truth point
  - `format_reward()` — formatting constraint reward
  - `LazySupervisedDataset` — custom dataset class for continual GUI data
  - `ContinualLearningArguments` — dataclass with KL beta, diversity, and dynamic beta configs
- **`trainer/grpo_trainer.py`** (1182 lines): Custom GRPO trainer extending HuggingFace TRL with:
  - Center point diversity bonus (variance-based)
  - Pairwise diversity bonus (Bhattacharyya distance)
  - Dynamic KL beta adjustment
  - Continual learning loss buffer
- **`vlm_modules/`**: Qwen2-VL module adapter, base VLM module
- **`trainer/grpo_config.py`**: GRPO-specific configuration

### Training Scripts
- `run_grpo.sh`: GRPO training with `--use_center_point_diversity`, `--use_pairwise_diversity`, `--center_point_diversity_weight 15` (alpha)
- `run_sft.sh`: SFT baseline training
- Both use `torchrun` for distributed training

### Evaluation
- `screenspotpro_test.py`: ScreenSpot-Pro evaluation (imports `process_utils` which is not in repo)
- `generate.py`: Model generation utility

## Portability Issues

1. **Setup path mismatch**: `setup.sh` references `src/open-r1-multimodal` but the actual code is under `src/gui-aif`. The setup script would fail out of the box.
2. **Hardcoded author paths**: `dataset.yaml` contains `/export/home2/n2409834c/GUI-AiF/example_training_json.json` — absolute author-local path.
3. **Template variables empty**: `run_grpo.sh` sets `CKPT_PATH=` (empty) and `DATA_PATH=/GUI-AiF/dataset.yaml` (absolute path).
4. **Likely GPU count bug**: `run_grpo.sh` sets `N_GPU_PER_NODE=54` — inconsistent with `CUDA_VISIBLE_DEVICES="1,2,3,4"` (4 GPUs). The variable appears unused but indicates poor cleanup.
5. **Missing imports**: `screenspotpro_test.py` imports `process_utils` which doesn't exist in the repository.
6. **Alpha=15 is hardcoded in run_grpo.sh**: The reviewer-debated alpha parameter (`center_point_diversity_weight=15`) is a command-line flag defaulting to 0.1 in code but set to 15 in the training script. This confirms the alpha discrepancy: the code default is 0.1 but the training config uses 15.

## Implementation Quality

The core algorithmic contribution (gaussian grounding rewards) is faithfully implemented:
- APR-iF (gaussian_plane_reward): computes reward as product of per-coordinate Gaussian probabilities with sigma = alpha * bbox_dimension
- ARR-iF (gaussian_point_reward): computes reward as probability under 2D Gaussian centered at ground-truth point
- Both use scipy.stats.multivariate_normal for the Gaussian computation
- Diversity bonuses are cleanly implemented in the trainer
- The continual learning arguments (dynamic_beta, diversity weights) are exposed as configurable flags

## Verdict

**Artifact Completeness: 5/10.** The core method is implemented (gaussian rewards, GRPO trainer, VLM modules), and the code is structurally sound. However, the repository has significant portability issues (wrong setup paths, hardcoded author paths, missing imports in evaluation) that prevent execution without author-side modifications. Points deducted for: (1) setup path mismatch, (2) hardcoded author paths, (3) missing evaluation imports, (4) empty template variables, (5) GPU count inconsistency. The method-to-code mapping is verifiable at the file level, but end-to-end reproducibility requires author cleanup.
