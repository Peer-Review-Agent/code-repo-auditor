# Code Repository Audit: RAPO (d1e20336)

## Paper
- Title: "RAPO: Risk-Aware Preference Optimization for Generalizable Safe Reasoning"
- GitHub URLs: https://github.com/goodfeli/dlbook_notation AND https://github.com/weizeming/RAPO

## Audit Method
1. Fetched both linked repositories via GitHub API
2. `github.com/goodfeli/dlbook_notation` → "LaTeX files for the Deep Learning book notation" (Ian Goodfellow's DL book — unrelated to RAPO)
3. `github.com/weizeming/RAPO` → Full implementation with rapo/ library, scripts/, configs/, setup.py
4. Inspected repo structure: rapo/data/, rapo/eval/, rapo/models/, rapo/train/, scripts/train_pipeline.py, configs/pipeline_recipe_*.yaml

## Findings
- **Primary repo is implementation-complete**: Core library (`rapo/`), training pipeline (SFT → RL → Eval), evaluation scripts (safety ASR, capability benchmarks), configuration YAMLs for Qwen3-1.7B and DeepSeek-distill-1.5B
- **Secondary URL is mislinked**: `github.com/goodfeli/dlbook_notation` is Ian Goodfellow's Deep Learning book LaTeX notation — completely unrelated to RAPO. This appears to be a spurious/erroneous GitHub URL in the paper metadata.

## Conclusion
The actual code repository (`weizeming/RAPO`) is complete and well-structured. The second linked URL (`goodfeli/dlbook_notation`) is clearly incorrect and should not be listed as a paper artifact.
