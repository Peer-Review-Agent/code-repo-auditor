# SPA (60121fd8) - Code Artifact Audit

**Paper:** SPA: A Simple but Tough-to-Beat Baseline for Knowledge Injection
**Repo:** https://github.com/Tangkexian/SPA
**Audit date:** 2026-04-26 ~04:00 UTC

## Method Summary

SPA uses 7 prompt templates (based on cognitive science learning strategies) to generate synthetic training data from small source corpora. The data is used for continued pretraining of LLMs. Claims to outperform RL-based (SEAL) and multi-stage prompting (EntiGraph, Active Reading) methods.

## What Is Present and Correct

1. **Seven prompt templates fully implemented** (`prompt_utils.py:1-97`): implications, key_concepts, teacher_style, discussions, case_studies, mind_map, qa_critical_thinking — matching the paper's three learning strategy categories (Concept Learning, Critical Thinking, Generative Learning).

2. **Data generation pipeline** (`generate_data.py`): Supports both vLLM (local) and OpenAI API generation, with resumable output, configurable sampling parameters, and per-prompt key selection. The `make_prompt()` function at line 14 correctly selects template type based on model family (OSS/instruct/base) and API mode.

3. **Training code** (`CPT_train.py`, `CPT_tokenize.py`): Standard HuggingFace Trainer-based continued pretraining with configurable hyperparameters, DeepSpeed support, and WandB logging.

4. **Shell scripts** provide complete end-to-end workflow: generation → tokenization → training.

5. **Sample data** (`data/squad_val_200.json`): A 200-article SQuAD subset for testing.

## What Is Missing

1. **No source datasets for QuALITY and MultiHop-RAG.** Only SQuAD has a sample JSON file. The `QuALITY` class imports from HuggingFace datasets, but no download scripts or data preparation code exists for MultiHop-RAG. Users must procure these datasets independently.

2. **No pre-generated synthetic data.** The paper reports results across multiple token budgets (scaling curves in Figure 1). Zero synthetic corpora are released. Reproducing requires running the full generation pipeline (OpenAI API credits or GPU hours for each of 7 prompts × K samples).

3. **No evaluation scripts.** There are zero scripts to evaluate trained models on SQuAD, QuALITY, or MultiHop-RAG. Missing:
   - SQuAD exact match / F1 evaluation
   - QuALITY accuracy evaluation
   - MultiHop-RAG answer evaluation
   - Any metric computation or result aggregation

4. **No trained model checkpoints.** The paper continues pretraining of LLaMA-3-8B and other models. No weights are released, no HuggingFace links.

5. **No baseline implementations.** SPA claims to outperform SEAL, PaST, EntiGraph, SoG, and Active Reading. None of these baseline implementations are included for comparison.

6. **No result reproduction scripts.** The exact hyperparameter configurations used to produce Tables 1-3 are not documented as launch configs. The `train.sh` script hardcodes one configuration, but the paper reports multiple configurations across benchmarks and token budgets.

7. **No figure generation scripts.** The scaling curves (Figures 1-2 in paper, `asserts/squad_scaling_curve.png` and `asserts/quality_scaling_curve.png`) cannot be regenerated — no data aggregation or plotting code exists.

## Assessment

This is a prompt-complete but evaluation-incomplete release. The seven prompt templates — which are the paper's core methodological contribution — are fully present and correctly implemented. However, the paper's central empirical claims cannot be independently verified because:

1. The source-to-synthetic pipeline requires expensive compute (OpenAI API or GPU generation for 7 prompts × 560 samples across 3 benchmarks), plus 8-GPU training.
2. The evaluation harness needed to measure downstream task performance is entirely absent.
3. No synthetic data or trained weights are released to shortcut verification.

This is a moderate reproducibility gap: the method is inspectable (the prompts are there), but the claimed results are not independently reproducible from the released artifacts.
