# Code Audit: MIGRASCOPE (ee71ef9e)

## Paper
"Revisiting RAG Retrievers: An Information Theoretic Benchmark"
arXiv: 2602.21553
GitHub: https://github.com/CapitalOne-Research/Migrascope

## Audit summary
The paper proposes MI-based metrics for RAG retriever analysis and ensemble search across 4 datasets. The repo implements the methodology correctly but defaults to toy-scale configs, omits several paper-evaluated retrievers, and requires significant infrastructure to reproduce any paper results.

## Findings

### Strengths
1. **Core methodology implemented correctly**: `migrascope/estimators.py` implements Gaussian MI via linear regression variance ratio (Equation: `MI = 0.5 * log(var(Y) / var(residual))`), non-Gaussian plugin MI via XGBoost/RandomForest, conditional MI, and exact + MC-permutation Shapley values.
2. **Attribution engine complete**: `migrascope/attribution.py` computes F_all (joint MI), marginals (conditional MI of each retriever given others), Shapley values, interaction matrix I(Y;Xi;Xj), and total MI per retriever.
3. **8 fusion methods implemented**: linear_z, prob_logop, logit_pool, noisy_or, rrf, borda, rra, rank_centrality — matching or exceeding the paper's ensemble search.
4. **Pseudo-GT estimation pipeline present**: `bench_post_run.py` implements LLM-based NLL scoring with label reinforcement and chunk alignment.
5. **Sample data included**: `data/hpqa/` contains HotpotQA corpus and QA JSON for testing.
6. **Config reproducibility**: `BenchmarkConfig.dump_snapshot()` saves JSON config for each run.

### Critical gaps between repo and paper claims

1. **Subsampling defaults produce toy results (high severity)**:
   - `SUBSAMPLE_CORPUS_SIZE = 1000` (paper used full corpus, typically 100k+ chunks)
   - `SUBSAMPLE_QA_SIZE = 10` (paper evaluated on 500-2000+ queries per dataset)
   - `NUM_TO_RETRIEVE = 800` note: the config says top-800 but the paper's analysis uses top-100 (`post_topK_threshold = 100`). This is consistent internally but the default 10-query run will not reproduce any paper figure or table.
   - These are the *committed default values* — a researcher who clones and runs `python bench_run.py` gets results on 10 queries, not the paper's experiments.

2. **Retrievers evaluated in the paper but missing from the code**:
   - README line 80-81: "The academic manuscript evaluates additional experimental graph construction techniques (e.g., LightRAG, GRAG-window, GRAG-global). The code for these extended methodologies is planned for inclusion in future releases."
   - The config's `RETRIEVER_LIST` only has 5 retrievers: GRAG-naive-semantic, QRAG-bge-m3, BM25, RAG-bge-m3, hippoRAG
   - The paper's full retriever analysis (e.g., redundancy/synergy figures involving LightRAG, GRAG-window, GRAG-global) cannot be reproduced from the current code.

3. **No pre-computed results or checkpoints committed**:
   - A researcher must run the full pipeline: (a) run all 5 retrievers on all 4 datasets, (b) run LLM-based pseudo-GT estimation, (c) run alignment, (d) run attribution, (e) run fusion search
   - Zero committed `results/result.jsonl`, zero attribution outputs, zero fusion search outputs
   - Zero retrieval checkpoints in `retriever_ckpts/`

4. **LLM dependency with fragile configuration**:
   - Pseudo-GT estimation (`bench_post_run.py`, `compute_MI.py` line 51-57) requires OpenAI-compatible `/v1/completions` endpoint with `echo=True`
   - README line 135 warns: "Standard /v1/chat/completions endpoints or some OpenAI models (like gpt-4o) that drop support for echo=True will crash"
   - The default model is `Meta-Llama-3-8B-Instruct` — requires local vLLM or similar serving infrastructure

5. **Dataset download not automated**:
   - Only HotpotQA sample in `data/hpqa/`
   - Full experiments require downloading and formatting 2WikiMultiHopQA, MuSiQue, TriviaQA per the schema in README lines 96-121
   - No download scripts or formatting scripts provided

## Verdict on reproducibility
The MIGRASCOPE repository provides a complete and correct implementation of the paper's MI-based retriever analysis methodology. However, the committed configuration defaults to toy-scale settings that cannot reproduce any paper figure, 3 retrievers the paper evaluates are not yet implemented, and the pipeline requires non-trivial LLM infrastructure with a specific API mode that many serving engines have deprecated. A determined researcher with the right infrastructure could reconstruct the paper's experiments by (a) adjusting config values, (b) downloading full datasets, (c) implementing the missing retrievers, and (d) provisioning an LLM server with legacy completions support. This is a moderate reproducibility gap: the code is real and correct, but the default path from clone to paper results is not supported.
