# Artifact Audit: RETO (6b87b08f)

## Paper
- **Title:** Robust and Efficient Tool Orchestration via Layered Execution Structures with Reflective Correction
- **arXiv:** 2602.18968
- **Repository:** https://github.com/anonymous-submission2026/RETO_code (mirror: https://github.com/orbit-913/RETO_code)

## Audit Method
Static inspection of the cloned repository. High-signal files examined in order: README, `dag_layer_prediction/` (core layer predictor), `tool_llama_client/` (inference/eval), `tool_llama_server/` (LLM serving). Did NOT run any code (policy compliance).

## Paper Claims vs. Repository Evidence

### Claim 1: "A lightweight and reusable orchestration component" (layer predictor)
**Paper (Section 3.2):** A BERT-based layer predictor trained on 5,035 tool sequences, using ordinal regression thresholds to assign tools to coarse layers.

**Repository evidence:**
- `dag_layer_prediction/dag_model.py` — The `ToolPlanningModel` (transformer encoder + linear head) correctly implements the described threshold-prediction architecture. ✓
- `dag_layer_prediction/dag_layer_predict.py` — Contains a full training loop with BCE logits loss, train/val/test split, model checkpointing. The argparse defaults (10 epochs, lr=1e-3, batch_size=64, seed=42) match common configurations. ✓
- **CRITICAL: `text_encoder.py` import is broken.** Line 16 of `dag_layer_predict.py` does `from text_encoder import Encoder`. No file `text_encoder.py` exists anywhere in the repository (`find /tmp/RETO_code -name "text_encoder.py"` returns nothing). This prevents the training script from executing. ✗
- **Training data absent.** The default `--data DTA_tool.layers.jsonl` is not committed. No example data, data generation scripts, or data format documentation is provided. ✗
- **No model weights.** No checkpoint, `.pth`, or pretrained weights file is committed. ✗

### Claim 2: "Schema-aware reflective correction mechanism"
**Paper (Section 3.3):** Detects and repairs execution-time errors locally without re-planning entire trajectories.

**Repository evidence:**
- `DAG_executor_llama.py` implements a multi-layered repair mechanism: `schema_gate_tool_calls` (parameter validation), `_should_attempt_repair` (error classification), `repair_tool_call_once` (LLM-based argument repair), and `_deterministic_fix_args_from_error` (one specific pattern: dropping unexpected keyword args). ✓ Partially present.
- However, the repair logic is largely generic JSON argument repair and LLM-retry patterns rather than a schema-aware mechanism that "detects and repairs errors locally" as described. The only schema awareness is parameter key filtering (lines in `_validate_args_level1`), not the deeper execution-strategy-level correction implied by the paper. ⚠ Overclaimed complexity.
- The server-side `_repair_json_trunc` (in `qwen_server.py`) handles malformed JSON output, which is basic output parsing, not the described correction mechanism. ⚠

### Claim 3: "Experimental results showing robust tool execution with reduced complexity"
**Paper (Table 1):** Qwen2.5-7B+RETO achieves 49.5 SoPR average on StableToolBench, above ToolLLaMA-7B+DFSDT (47.1) and GPT-3.5-0613+ReAct (45.4).

**Repository evidence:**
- Inference scripts: `run_qa_pipeline_multithread_qwen_DAG.sh` launches the DAG pipeline. ✓
- Evaluation scripts: `run_pass_rate.sh`, `run_preference.sh`, `run_convert_answer.sh` exist for StableToolBench evaluation. ✓
- However, reproducing the exact Table 1 result requires: (a) training the layer predictor with missing `text_encoder.py` and missing `DTA_tool.layers.jsonl`, (b) deploying a Qwen2.5-7B server, (c) running the DAG pipeline, (d) running evaluation. No single config or shell script automates this chain. ✗
- The README's "Framework infra running example" shows only inference, not training or evaluation. ✗

## Assessment
The repository is a credible partial implementation of RETO's core idea (layered execution) and demonstrates the DAG-based execution flow with repair mechanisms. The training pipeline for the layer predictor exists but is broken by a missing import (`text_encoder.py`). Training data and model weights are absent. The reflective correction is implemented at a simpler level than described. Reproducing the paper's central empirical claim (Table 1: 49.5 SoPR) is not feasible from the released artifacts alone.
