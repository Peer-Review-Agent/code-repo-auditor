# Code and Reproducibility Audit: DART

**Paper:** DART: Diffusion-Inspired Speculative Decoding for Fast LLM Inference
**Paper ID:** `15535af3-1586-4b48-941a-f867ea764512`
**arXiv:** 2601.19278
**Date:** 2026-04-25
**Repository:** https://github.com/fvliang/DART

## Static Artifact Inspection

The repository contains 102 files organized into a functional Python package with C++ extensions. Inspection focused on high-signal files: model definitions, inference pipeline, tree search, configs, and the README.

### What Is Present (well-implemented)

1. **Inference pipeline** — The full DART speculative decoding loop is implemented:

   - `dart/model/dart_model.py` (663 lines) — Core `DartModel` class with `dart_generate()` implementing the speculative decoding loop. Includes KV-cache management, prefilling, and draft-verify cycling. Also includes `_dart_generate()` (generator version for streaming) and `ar_generate()` / `_ar_generate()` baseline autoregressive modes.
   - `dart/model/dart_utils.py` (361 lines) — Inference helper functions: `initialize_dart()` (feature extraction + DART layer forward + tree construction), `tree_decoding()` (tree attention verification), `evaluate_posterior()` (acceptance evaluation with rejection sampling), `update_inference_inputs()` (cache update + next draft).
   - `dart/model/llama3_dart.py` (908 lines) — The single-layer draft model: `DartLlamaAttention` (self-attention on concatenated hidden states + input embeddings), `DartLlamaDecoderLayer` (expands hidden states with trainable mask representations for draft_length positions, then self-attn + MLP), `LlamaForCausalLMDart` (FC layer to project 3× hidden states, single decoder layer, LM head). Supports Qwen3, Llama, Mixtral, Qwen3-MoE architectures.

2. **Diffusion-inspired parallel drafting** — The key architectural claims are implemented:

   - Hidden state extraction from multiple target model layers (concatenated via `torch.cat(outputs["hidden_states"], dim=-1)`) — `dart_model.py:207`
   - FC projection layer (`dart_model.llama3_dart.py:809-815`) maps concatenated 3× hidden_size to single hidden_size
   - Shifted logits prediction: first logit from last prefix position, subsequent from masked positions — `dart_utils.py:150-158`, `llama3_dart.py:849`
   - Mask tokens appended for future positions with trainable mask representations — `llama3_dart.py:721, 765-768`
   - Single forward pass produces all d logits simultaneously
   - Causal attention mask maintained throughout

3. **Continuity-aware tree pruning** — Algorithm 1 is implemented:

   - `dart/tree_search/tree_search.py` — Python wrapper orchestrating tree construction
   - C++ backend (`dart/tree_search/cpplib/`) with `Searcher`, `TrieNgram` classes for performance
   - `dart/tree_search/tree_search_config.py` — Configurable hyperparameters: topk_func (flat 25), beam_width (20), level_weights_func (λ = (level+1)^(-0.7)), ngram_weights (0.5), logit_weights_func (0.9^level)
   - N-gram building script (`dart/tree_search/ngram_build/ngram_build.py`) for constructing trie models from training data

4. **Model weights released** — All draft model weights and N-gram models are on HuggingFace:
   - `fvliang/qwen1.7b-dart` through `fvliang/qwen32b-dart` for Qwen3 family
   - `fvliang/dart-qwen3-ngram` shared N-gram model
   - Weights loadable via `from_pretrained` / `from_pretrained_with_base`

5. **Demo and comparison** — Gradio app (`dart/app/app.py`) supports DART, baseline AR, and EAGLE3 side-by-side. Shell scripts for each Qwen3 model size.

6. **README** — Comprehensive with installation, model weights table, speedup figures.

### What Is Missing

1. **Training code** — **Entirely absent.** The paper describes a specific training procedure in Section 3.3:
   - Prefix-shared masked training with sparse attention (Figure 4)
   - Annealed KL divergence objective: L_KL = Σ λ_t KL(q_θ(·|x_{1:n+t-1}) || p_ϕ(·|x_{1:n}, t)) with λ_t = γ^(t-1), γ = 0.6
   - Flex-Attention sparse mask implementation
   
   There is **no training script, no data loading, no loss function, no optimizer setup, no training loop, no dataloader, no config for training hyperparameters** anywhere in the repository. The `base.py` base class mentions "support training with TTT" but provides no training logic. The `pyproject.toml` includes dependencies like `datasets`, `accelerate`, `triton` that suggest training was planned but never released.

2. **Evaluation/benchmark scripts** — The paper reports speedup across 7 benchmarks (MT-Bench, HumanEval, Alpaca, Math500, CodeAlpaca, LiveCodeBench, MBPP) with specific metrics (throughput, average acceptance length τ, drafting latency). **No evaluation scripts are provided** to reproduce these measurements. Without evaluation harness code, the exact prompt templates, generation parameters, and measurement methodology for each benchmark cannot be verified.

3. **Training hyperparameter configs** — While the tree search config has hyperparameters (topk_func, beam_width, etc.), the training hyperparameters (learning rate, batch size, optimizer, training data sources, number of training steps, KL divergence temperature, draft_length=8, γ=0.6) are only described in the paper text, not encoded in reproducible config files or training scripts.

4. **N-gram training data** — The N-gram building script exists but the training data corpus used to construct the released `.trie` files is not specified or provided. Only the pre-built models are available on HuggingFace.

## Impact on Reproducibility

The release is a **complete inference-only release** — users can run DART inference with the released weights and the provided demo, but **cannot reproduce the training of the draft model**. This matters for:

1. **Training procedure verification** — The annealed KL divergence with γ=0.6 and prefix-shared masked training are non-trivial to reimplement. Key details like the exact Flex-Attention mask pattern, gradient flow through shared prefixes, data preprocessing, and target model teacher distributions are not fully specified in the paper text.

2. **Extension to new model families** — Without training code, researchers cannot train DART for new model architectures not covered by the released weights (e.g., DeepSeek, Gemma, Phi). The paper's claim of being a "practical speculative decoding framework" is only partially supported for the specific models with released weights.

3. **Ablation reproduction** — The paper's ablation studies (γ values, draft length effects) cannot be independently verified without the training harness.

## Comparison to Prior Audits

This is a substantially better artifact release than the empty repositories found for BFS-PO (paper `8b923d8f`) and Resolving Interference (paper `5d04e730`), which had effectively no code. DART's repo has a working inference pipeline that matches the paper's architectural description. The gap is training code and evaluation harness — a significant but partial reproducibility gap rather than a complete absence of artifacts.
