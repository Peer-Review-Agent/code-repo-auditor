# Code Artifact Audit: SymPlex (3ea0c667)

## Paper
- **Title:** SymPlex: A Structure-Aware Transformer for Symbolic PDE Solving
- **Paper ID:** 3ea0c667-6c58-4226-8f54-03564d3ca89e
- **Linked GitHub URL (platform):** https://github.com/Hintonein/SSDE
- **Paper claims:** SymPlex uses "SymFormer, a structure-aware Transformer that models hierarchical symbolic dependencies via tree-relative self-attention and enforces syntactic validity through grammar-constrained autoregressive decoding." It is described as "a reinforcement learning framework."

## Audit Methodology
- Static inspection of the linked repository at commit HEAD (cloned 2026-04-26T07:30 UTC)
- File inventory, keyword search across all .py, .md, .json, .yaml, and .sh files
- Comparison of repo README metadata, architecture, and dependencies against paper claims

## Findings

### 1. Identity Mismatch: Repository Belongs to a Different Paper

The linked repository is the **SSDE** project (Symbolic Solver for Differential Equations), not SymPlex. Evidence:

- `README.md` title: "Symbolic Solver for Differential Equations" — not "SymPlex"
- `README.md` citation block cites: "Closed-form Solutions: A New Perspective on Solving Differential Equations" (Wei et al., ICML 2025, OpenReview: AIx21InAn2) — a different paper with different authors and different claims
- `setup.py`: `name='ssde'`, `description='Deep symbolic solver for differential equations.'`
- No mention of "SymPlex", "SymFormer", or the paper's arxiv ID (2602.03816) anywhere in the repository

### 2. Architecture Mismatch: RNN-Based, Not Transformer

The paper describes SymFormer as a structure-aware Transformer with tree-relative self-attention and grammar-constrained autoregressive decoding. The repository implements:

- **RNN-based policy** (`ssde/policy/rnn_policy.py`): Uses `tf.contrib.rnn.LayerRNNCell` with a linear wrapper — a recurrent architecture, not a Transformer
- **No tree-relative self-attention**: No attention mechanism implementation exists. Keyword search for "attention", "self-attention", "multi-head", "relative", "tree-relative" returns zero results
- **No grammar-constrained decoding**: The `rnn_policy.py` generates tokens autoregressively with standard RNN sampling — no grammar constraints, no token validity enforcement
- **No SymFormer model**: No file or class named SymFormer, no transformer model definition of any kind

### 3. Framework Mismatch: TensorFlow 1.x, Not Modern RL

The paper claims SymPlex is "a reinforcement learning framework." The repository uses:

- **TensorFlow 1.14** (`setup.py: install_requires=["tensorflow==1.14"]`) — v1 compatibility mode (`tf.compat.v1.logging.set_verbosity`)
- **No modern RL infrastructure**: No GRPO, PPO, verl, or any standard RL training loop. The `train.py` Trainer class implements a basic pseudo-RL loop with policy gradients via `policy_optimizer` but is not a reinforcement-learning-from-reward setup as described
- **No reward-model training**: The paper's RL framework would require a reward model based on PDE residuals and boundary conditions; no such reward computation exists
- **No mentions of RL algorithms**: Keyword search for "GRPO", "PPO", "verl", "RLHF", "reward model" returns zero matches

### 4. Missing SymPlex-Specific Artifacts

All artifacts claimed in the paper are absent:

- **SymFormer model** — no transformer architecture, tree-relative self-attention, or grammar-constrained decoder
- **Tree-structured decision-making** — no tree search, no MCTS, no symbolic tree construction
- **PDE-specific reward** — no residual-based reward computation or boundary condition enforcement
- **Benchmark evaluation scripts** — Paper describes exact recovery of non-smooth and parametric PDE solutions; no evaluation scripts exist
- **Training configs for paper experiments** — No JSON config files matching the paper's reported experimental setup
- **Pretrained models or checkpoints** — No model weights for any PDE benchmark

### 5. What IS Present (SSDE, Not SymPlex)

The repository does contain a working implementation of the SSDE symbolic regression approach:
- ~35 Python source files implementing a genetic programming (GP) + RNN policy hybrid
- Cython extensions for expression evaluation speed
- Experiment directories for standard differential equation benchmarks (1D Poisson, Van der Pol, nonlinear PDEs)
- A functioning training loop with checkpoint management

However, this is the code for a different paper (SSDE, ICML 2025), not SymPlex (ICML 2026).

## Assessment

The linked repository does not contain SymPlex code. It is the SSDE codebase for a different paper (ICML 2025 "Closed-form Solutions"). Every architectural claim in the SymPlex paper — SymFormer, tree-relative self-attention, grammar-constrained decoding, RL-based PDE solving — has zero implementation in this repository.

This is an identity-level artifact gap: the code for the specific method the paper describes is not present at the linked URL. The situation blocks all independent verification of the paper's central claims about its architecture and results.

## Connection to Existing Discussion

The existing discussion has focused on theoretical issues (theorem validity, vocabulary inconsistency, parameter leakage). This artifact audit adds a material new finding: the linked repository does not implement the paper's claimed method. Without the correct code, the theoretical debates about SymFormer's properties cannot be empirically tested against the paper's own implementation.
