## Audit Method
1. Fetched paper PDF and read abstract, introduction, and methodology sections (pages 1-4)
2. Paper explicitly states on page 1: "Codebase and data: https://github.com/EstebanGarces/human_vs_machine"
3. Attempted to fetch the repository via GitHub API: `GET /repos/EstebanGarces/human_vs_machine` returns 404 Not Found
4. Searched for the repository via GitHub search API: `GET /search/repositories?q=human_vs_machine+user:EstebanGarces` — no results
5. Fetched the user's public repos: `GET /users/EstebanGarces/repos` — return indicates no public repositories

## Findings
The linked GitHub repository `https://github.com/EstebanGarces/human_vs_machine` does not exist. The GitHub user account `EstebanGarces` exists but has zero public repositories. This means:
- The analysis code for the 1.8 million text samples across 8 language models is unavailable
- The decoding strategy implementations (top-k, nucleus sampling, contrastive search across 53 hyperparameter configurations) cannot be inspected
- The detection classifier training/evaluation code is absent
- The data preprocessing and statistical analysis pipeline is unreproducible
- Results across GPT2-XL, Qwen2-7B, Mamba, RWKV, and other architectures cannot be verified

## Conclusion
The paper's central empirical contribution (a large-scale study of the truncation blind spot across 1.8M texts) is entirely unverifiable from the linked artifact. The GitHub URL provided in the paper body returns 404.
