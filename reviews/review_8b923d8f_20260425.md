# Code and Reproducibility Audit: BFS-PO

**Paper:** BFS-PO: Best-First Search for Large Reasoning Models  
**Paper ID:** `8b923d8f-5d39-4b5a-8d70-1ed0cd54ad4c`  
**arXiv:** 2602.14917  
**Date:** 2026-04-25

## Finding: Repository Contains Only a Placeholder README and a Figure

The paper links to `https://github.com/aimagelab/BFS-PO` as its code repository.

Static inspection reveals:
- `README.md` — 8 lines, explicitly states: "This repository will provide the official implementation and associated experimental resources for BFS-PO... Additional documentation and updates will be made available here." (future tense)
- `bfspo.png` — A single framework overview figure
- **No code files, no training scripts, no evaluation scripts, no configs, no requirements, no data loaders.**

The paper tarball also contains only LaTeX source (`.tex`, `.sty`, `.bib`, `.bbl`) with no supplementary code.

## Impact on Reproducibility

BFS-PO is an RL algorithm with a non-trivial training procedure:

1. **Best-First Search exploration strategy** — The core contribution is a backtracking mechanism based on "maximum entropy nodes." Without code, the exact node-selection heuristic, the backtracking condition, and the tree-pruning logic cannot be verified.
2. **Training loop** — The paper uses GRPO/DAPO as the base RL algorithm and modifies the exploration step. Reproducing this requires integrating BFS-PO into an existing RL harness (e.g., verl, TRL, OpenRLHF), but the integration points are not specified.
3. **Benchmark evaluation** — The paper claims improvements on "different benchmarks and base LRMs." Without evaluation scripts and exact prompts, these claims cannot be independently validated.
4. **Hyperparameters** — Key hyperparameters such as the maximum entropy threshold, the backtracking depth, and the reward shaping are not fully tabulated.

## Conclusion

The repository is a placeholder with no runnable code. This blocks verification of the central claim that BFS-PO "can simultaneously increase the LRM accuracy and shorten its answers." The absence is particularly consequential because the method involves a **novel RL exploration strategy** that is not straightforward to reimplement from the paper description alone.
