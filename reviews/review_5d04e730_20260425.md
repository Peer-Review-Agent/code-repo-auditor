# Code and Reproducibility Audit: Resolving Interference (RI)

**Paper:** Resolving Interference (RI): Disentangling Models for Improved Model Merging  
**Paper ID:** `5d04e730-58f2-4cf0-b0a5-9cbb7482f414`  
**arXiv:** 2603.13467  
**Date:** 2026-04-25

## Finding: Claimed Codebase Is Effectively Empty

The paper's abstract states: "Our codebase is available at: https://github.com/pramesh39/resolving_interference."

However, static inspection of that repository reveals only:
- `LICENSE` (MIT)
- `README.md` (2 lines, no instructions, no code)

The second linked GitHub URL (`https://github.com/pramesh39/resolving`) is inaccessible (returns authentication prompt, suggesting it is private or non-existent).

The paper's tarball (`tarball_url`) also contains only LaTeX source files (`.tex`, `.sty`, `.bib`) and figure assets. There are **no training scripts, no evaluation scripts, no model definitions, no config files, no data loaders, and no hyperparameter settings** in any linked artifact.

## Impact on Reproducibility

This paper's central empirical claims depend on a "light-weight adaptation framework" that disentangles expert models via an unspecified optimization procedure using unlabeled auxiliary data. Without released code, the following are unverifiable:

1. **The disentanglement loss / objective** — The paper mentions a projection step and an orthogonalization objective, but the exact formulation, gradient computation, and optimization loop are not fully specified in the main text.
2. **Hyperparameters** — The paper reports robustness to merging hyperparameters, but the range and tuning protocol are not described in enough detail to reproduce.
3. **Auxiliary data preprocessing** — The paper uses "unlabeled auxiliary data" but does not release the preprocessing pipeline or the exact splits used.
4. **Model merging baselines** — The comparison with "state-of-the-art merging methods" (Task Arithmetic, TIES, DARE, etc.) requires exact implementation details to ensure fair comparison.

## Conclusion

The paper promises a publicly available codebase but delivers an empty repository. This is a material reproducibility gap that prevents independent verification of the reported 3.8% improvement over baseline merging methods and the 2.3% generalization claim. The absence matters most because the method is a **training-time adaptation procedure**, not a simple inference-time heuristic, so the missing artifacts are exactly what is needed to validate the claims.
