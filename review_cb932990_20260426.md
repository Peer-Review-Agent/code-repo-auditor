# Code Audit: SurrogateSHAP (cb932990)

## Paper
- **Title:** SurrogateSHAP: Training-Free Contributor Attribution for Text-to-Image (T2I) Models
- **arXiv:** 2601.22276
- **Paper ID:** cb932990-d35d-403b-9d95-aa76ff3fa888

## Audit Summary

### Claim
The paper proposes SurrogateSHAP, a retraining-free framework for contributor attribution in T2I diffusion models, using gradient-boosted trees to approximate the utility function and derive Shapley values analytically. The method is evaluated on 3 tasks: (i) FID/IS on DDPM-CFG CIFAR-20, (ii) aesthetic quality on Stable Diffusion Post-Impressionist artworks, (iii) product diversity on FLUX.1 Fashion-Product data.

### Verification
- **DAS repo (Jinxu-Lin/DAS):** Confirmed via web fetch — README states it implements "Diffusion Attribution Score (DAS)" accepted at ICLR 2025. Contains DAS-specific code for CIFAR2/CIFAR10/ArtBench/CelebA — no SurrogateSHAP implementation.
- **Efficient Framework repo (suinleelab/...):** Confirmed via web fetch — README states it implements "An Efficient Framework for Crediting Data Contributors of Diffusion Models" (ICLR 2025). Contains LDS-based code — no SurrogateSHAP implementation.
- **Remaining 6 repos:** All are external tools/models/baselines, none related to SurrogateSHAP.

### Tarball Contents
65 files total: LaTeX source (23 .tex files), figures (27 .pdf + 8 .png), style files (4 .sty), bibliography (1 .bib, 1 .bst), and metadata (1 .json). **Zero code files (no .py, .sh, .ipynb, .yaml, requirements.txt, or Dockerfile).**

### Finding
**The paper lists 8 GitHub URLs, but none of them contain the SurrogateSHAP implementation.** There is no repository for the paper's method. The tarball is purely LaTeX source. This means:

1. The SurrogateSHAP method itself cannot be reproduced from the provided artifacts.
2. The training-free attribution pipeline using gradient-boosted trees + analytical Shapley derivation has no reference implementation.
3. The 3-task evaluation (DDPM-CFG CIFAR-20, Stable Diffusion aesthetics, FLUX.1 Fashion-Product diversity) cannot be replicated.
4. The comparisons against DAS, D-TRAK, LDS baselines rely on metric implementations (FID, IS) that exist in external repos, but the SurrogateSHAP pipeline connecting them is missing.
5. The counterfactual clinical image spurious correlation experiment has no reproduction path.

### Impact on Reproducibility Assessment
This is not a minor omission — the paper's core method has no implementation artifact. The 8 GitHub URLs create the misleading appearance of code availability while actually supplying only dependencies and evaluation tools. This directly blocks verification of the paper's central empirical claims.
