# Code Audit: Semi-knockoffs (8e32c11f)

## Paper
- **Title:** Semi-knockoffs: a model-agnostic conditional independence testing method with finite-sample guarantees
- **Paper ID:** 8e32c11f-a037-4ae0-933a-146232c111a2

## Audit Summary

### Claim
The paper proposes Semi-knockoffs, a conditional independence testing method that accommodates any pre-trained model, avoids train-test split, provides valid p-values and FDR control in high-dimensional settings, and introduces two theoretical results (stability for regularized models with null feature, and double-robustness).

### GitHub URLs Listed (2 total)
1. `https://github.com/AngelReyero/loss_based_KO` — **Confirmed: this IS the author's implementation.** README states "Semi-knockoffs: a model-agnostic conditional independence testing method with finite-sample guarantees" — matching the paper title.
2. `https://github.com/AngelReyero/loss` — **Broken link: returns HTTP 404.** This URL is inaccessible.

### Positive Findings (from `loss_based_KO` repo)

**Repository health:**
- 34 commits, indicating sustained development effort
- Contains `src/` directory with experiment replication code
- Contains `results/` directory with "experimental figures, along with the CSV files needed to reproduce them" (per README)
- Languages: Jupyter Notebook (80.1%), Python (19.9%)

**Reproducibility support:**
- README explicitly states "the `src` folder contains the code to replicate the experiments"
- README acknowledges `results` folder provides figures and CSVs for reproduction
- Documents baseline dependencies: "For comparisons with Knockoffs, dCRT, and LOCO, we use the implementations from Hidimstats, and for HRT, we rely on pyHRT"

**Gaps:**
- No explicit `requirements.txt` or `environment.yml` visible in the top-level file listing
- No explicit training/evaluation entrypoints documented in the README (no `python train.py` style instructions)
- The second listed GitHub URL (`AngelReyero/loss`) is a dead link (404)

### Finding
**The primary repository (`AngelReyero/loss_based_KO`) is a legitimate, functional implementation artifact.** It contains source code, results, and explicit claims of experiment reproducibility. This is a positive finding — the paper's core method has an accessible implementation.

However, the second listed URL is broken (404), and the README could be improved with explicit dependency installation instructions and entrypoint commands to lower the barrier to reproduction.

### Impact on Reproducibility
The existence of a working implementation repo with source code and results makes the Semi-knockoffs method reproducible in principle. The missing requirements file and ambiguous entrypoints are minor papercuts, not blockers. The broken second URL should be fixed or removed from the paper metadata. Overall, this is a well-supported artifact audit.
